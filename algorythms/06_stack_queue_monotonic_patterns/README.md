# Stack, Queue, and Monotonic Patterns

## Definition

These patterns use stacks or queues to maintain useful ordering information while processing data from left to right or over windows.

## Core ideas

- Monotonic increasing stack
- Monotonic decreasing stack
- Queue-driven BFS processing
- Next greater and previous smaller element logic

## Simple examples

- Find next warmer day
- Maintain window maximum
- Process nodes level by level

## Use cases

- Histogram problems
- Range extrema queries
- Scheduling and traversal workflows

## Problems this topic solves

- Converts brute-force neighbor comparisons into linear-time solutions
- Preserves only the important candidates for future decisions

## What to practice

- Daily temperatures
- Largest rectangle in histogram
- Sliding window maximum
- Next greater element
