# Stacks

## Definition

A stack is a Last In, First Out (LIFO) data structure where the most recently added item is the first one removed.

## Core ideas

- Push, pop, and peek
- LIFO ordering
- Can be built with arrays or linked lists

## Simple examples

- Undo history
- Balanced parentheses checker
- Expression evaluation

## Use cases

- Function call management
- Backtracking
- Parsing expressions
- Monotonic stack problems

## Problems this topic solves

- Tracks recent state efficiently
- Supports nested structure validation
- Simplifies reversal-like workflows

## What to practice

- Valid parentheses
- Min stack
- Daily temperatures
- Evaluate postfix expressions
