# Linked Lists

## Definition

A linked list is a sequence of nodes where each node stores a value and a reference to the next node, and sometimes the previous node.

## Core ideas

- Singly vs doubly linked lists
- Head and tail pointers
- Pointer manipulation
- Insertions and deletions without shifting elements

## Simple examples

- Insert a node at the front
- Delete a node by value
- Reverse a linked list
- Detect a cycle

## Use cases

- Frequent insertions and deletions
- Implementing stacks, queues, and adjacency lists
- Systems with unknown or changing collection sizes

## Problems this topic solves

- Avoids expensive shifting required in arrays
- Models dynamic ordered data efficiently

## What to practice

- Reverse linked list
- Merge two sorted lists
- Remove nth node from end
- Find middle node
- Detect cycle with fast and slow pointers

## Suggested folder pattern

For each problem, create a numbered folder like:

- `01_reverse_linked_list`
- `02_merge_two_sorted_lists`
- `03_linked_list_cycle`

Inside each one, keep:

- `problem.md`
- `solution.py`
- `solution.js`
- `solution.ts`
- `solution.rb`
