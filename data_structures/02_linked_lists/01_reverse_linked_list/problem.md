# Reverse Linked List

## Difficulty

Easy

## Prompt

Given the head of a singly linked list, reverse the list and return the new head.

## Example

Input:

`1 -> 2 -> 3 -> 4 -> 5`

Output:

`5 -> 4 -> 3 -> 2 -> 1`

## Follow-up

- Can you solve it iteratively?
- Can you solve it recursively?
- What is the time and space complexity?

## Concepts trained

- Pointer manipulation
- Iterative thinking
- Recursive thinking
- Space complexity analysis

## Notes

This is one of the best starter linked list problems because it forces you to understand how node references change over time.
