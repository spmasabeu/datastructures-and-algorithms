# Big O and Recursion

## Definition

Big O describes how runtime or memory usage grows as input size grows. Recursion solves a problem by reducing it to smaller versions of itself.

## Core ideas

- Time complexity
- Space complexity
- Base case and recursive case
- Call stack behavior

## Simple examples

- Factorial
- Fibonacci
- Recursive tree traversal

## Use cases

- Comparing algorithm efficiency
- Breaking large problems into smaller ones
- Reasoning about scalability

## Problems this topic solves

- Helps choose the right algorithm
- Builds the mental model needed for trees, divide-and-conquer, and backtracking

## What to practice

- Compute power recursively
- Generate subsets recursively
- Analyze best, average, and worst-case complexity
